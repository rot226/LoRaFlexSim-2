# Mini-charte UI — Terminologie et unités

Cette mini-charte s’applique aux libellés UI (widgets, indicateurs, titres de graphiques, axes) du tableau de bord.

## 1) Terminologie stable (obligatoire)

Utiliser exactement ces termes pour les métriques clés :

- `Packet Delivery Ratio (PDR)`
- `Transmission Energy`
- `Mean Delay`
- `Throughput`

## 2) Unités SI homogènes

Utiliser systématiquement les unités suivantes (et cette casse) dans les libellés :

- `s`
- `dBm`
- `dB`
- `J`
- `bit/s`

Éviter les variantes incohérentes (ex. `bps`, `sec`, `Db`, etc.).

## 3) Style des labels

- Employer **Title Case** pour les libellés visibles.
- Conserver une convention unique pour les unités : `Metric Name (unit)`.
- Garder la même ponctuation pour les parenthèses et séparateurs.

## 4) Abréviations critiques

À la première occurrence visible, expliciter l’abréviation :

- `Packet Delivery Ratio (PDR)`
- `Spreading Factor (SF)`
- `Signal-to-Noise-and-Interference Ratio (SNIR)`

Ensuite, l’abréviation seule est autorisée si le contexte reste clair.
