# Debug / Sanity checks

> [!TIP]
> **What is this folder for?** Documenting quick consistency checks on CSV results, especially around SNIR, PDR, and distributions.
>
> **When should it be used?** After a simulation campaign, before detailed analysis, or whenever a result looks suspicious.
>
> **Minimal command** `python scripts/sanity_checks.py`
>
> **Produced outputs** `WARN`/`FAIL` messages in the console to flag inconsistencies detected in results.

## 1. Folder objective

The `scripts/sanity_checks.py` script quickly checks the consistency of CSV outputs (SNIR ON/OFF, PDR, distributions) and reports anomalies as **WARN** or **FAIL**.

## 2. Prerequisites

### Local policy aligned with the main README

- **Primary documented OS: Windows 11**.
- **Documented shell: PowerShell**.
- **Execution directory: repository root**.
- **Recommended Python version: 3.11**.
- **Packaging support: Python 3.11 to 3.12**.
- **Recommended standard installation:** `python -m pip install -e . --no-build-isolation` after activating the venv.
- **`PYTHONPATH=src` is not required** for these checks in standard usage.
- **`cmd.exe` is not the primary documentation target**; use PowerShell.

## 3. Minimal scenario

From the **repository root** in **PowerShell**:

```powershell
python scripts/sanity_checks.py
```

## 4. Run command

### Usage

From the **repository root** in **PowerShell**:

```powershell
# Analyse par défaut (tous les CSV sous results/)
python scripts/sanity_checks.py

# Cibler un dossier de résultats spécifique
python scripts/sanity_checks.py results/step1

# Ajuster les seuils
python scripts/sanity_checks.py results/step1 --epsilon 0.02 --large-nodes 150 --pdr-der-threshold 0.999

# Échouer si un WARN est détecté
python scripts/sanity_checks.py results/step1 --fail-on-warn
```

## 5. Aggregation

There is no dedicated aggregation: this workflow directly inspects CSV files already produced in `results/`.

## 6. Plots

No plots are generated by this script: output is textual and intended for quick validation.

## 7. Report

No dedicated report is produced: information is displayed in the console for immediate diagnosis.

## 8. Detailed figures and advanced options

### Verified behavior

- Compares SNIR **ON** vs **OFF** on **PDR / throughput / collisions** (Δ > ε).
- Warns if **PDR/DER > 0.999** for a high number of nodes.
- Checks for **non-zero variance** in **SF/SNR/SNIR/collisions** distributions (JSON histograms).
- Warns if **Jain == 1.0** on all rows.

### Windows 11 notes

- Use `python` or `py -3.11` depending on your setup (`py -3.11 scripts/sanity_checks.py`).
- Windows paths (`C:\...`) are accepted for `results/` if needed.
- If you are in offline/fallback mode, first activate the venv and use the script documented in `README.md` if necessary.
